# 装饰器模式
* 装饰器模式主要对现有的类对象进行包裹和封装，以期望在不改变类对象及其类定义的情况下，为对象添加额外功能。是一种对象结构型模式。需要注意的是，该过程是通过调用被包裹之后的对象完成功能添加的，而不是直接修改现有对象的行为，相当于增加了中间层  

## 优缺点
* 优点：  
1、相比较于类的继承来扩展功能，对对象进行包裹更加的灵活；  
2、装饰类和被装饰类相互独立，耦合度较低；  
* 缺点：  
1、没有继承结构清晰；  
2、包裹层数较多时，难以理解和管理；  

**注意：个人觉得装饰器模式和代理模式乍一看很像，装饰器模式主要关注于给一个对象（被装饰对象）扩展可能需要的多个功能（具体的装饰器），而代理模式则注重于将某个增强功能应用到多个对象（被代理对象）上**
